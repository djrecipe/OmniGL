GLint OmniOGL::OGLAttributes::color
private

A shader program attribute which supplies one color (r,g,b,a) for every vertex sent. Note that color data will only be used if textures are disabled.

This attribute is bound to a buffer object and colors are sent to the graphics card via calls to glBindBuffer() and glBufferData().

GLint OmniOGL::OGLAttributes::normal
private

A shader program attribute for transmitting the normal vector on a per-vertex basis.

During the first pass, one normal is computed for every triangle (every three vertices), then the computed normal is assigned to each vertex of the triangle so each vertex has its own normal. On the second pass, any normals which share a vertex (i.e. two adjacent triangles would share two vertices and therefore two normals) are then averaged together.

GLint OmniOGL::OGLAttributes::position
private

A shader program attribute used to update the vertices of any object-to-be-rendered. This attribute is perhaps the most crucial as it directly supplies the vertices of the polygons which are to-be-drawn.

This attribute is bound to a buffer object (probably object index zero) and vertices are sent to the graphics card via calls to glBindBuffer() and glBufferData().

GLint OmniOGL::OGLAttributes::srt[4]
private

A shader program attribute for transmitting the combined translation / rotation / scale matrix. This matrix is updated on an as-needed basis, but it is updated often, so it is essential to transfer the data via an attribute.

This matrix is calculated by multiplying translation matrix * rotation matrix * scale matrix. Note that matrix multiplication is non-communative, so the order in which the matrices are multiplied will have an impact on the result. When instanced rendering is being utilized, this attribute is updated once per object.
