GLint OmniOGL::TextureMaster::MakeTexture ( std::string  file_in,
bool  repeat 
)
staticprivate

A static method which takes the path to a picture file as a parameter, uses the SOIL library to load the picture into local memory, then generates, activates, and binds a texture in the shader program. The index of the texture is stored in a static member so that the texture can be referenced later.

See Also
RenObj::tex, RenObj::SetTexture()

Parameters
file_inA string representing a file path for the texture image which is desired. If the file path has already been bound to a texture then the appropriate texture index will be used and no new texture will be bound.
repeatA boolean value which determines whether or not the texture is tiled (repeated) or stretched.
Returns
Provided there were no errors, returns the texture index used, regardless if a new texture was bound or not. Errors will result in a return value of -1.

std::string OmniOGL::TextureMaster::filenames
staticprivate

An array of paths of the picture files which have already been loaded into a texture. These paths are stored each time a texture is created so-as to prevent accidentally assigning the same file to multiple texture "slots".

See Also
RenObj::MakeTexture()

For simplicity's sake, the size of this array is static. In the future, a more flexible implementation might size this array according to the actual maximum number of textures supported by the graphics card of the system on which the program is run.
